# 供应商收藏

## 目标与范围

在供应商较多时快速标记常用项，并可一键只看收藏项。本特性仅改前端列表体验，不修改供应商结构或后端数据库。

## 用户行为

- 每张 Provider 卡片左上角显示星标按钮；点击后切换收藏状态。
- 「仅收藏」快捷筛选紧邻「仅显示可用」，可与搜索、可用状态和模型筛选叠加。
- 收藏按应用分别保存在本机，例如 Codex 与 Claude 的收藏互不影响。
- 收藏状态在应用重启后保留；存储不可用或数据损坏时回退为空收藏，不影响供应商列表。

## 关键决策

- 使用 `localStorage` 键 `cc-switch-provider-favorites:v1:<appId>`，值为去重后的供应商 ID 数组。
- 不把收藏写入 Provider 数据模型，避免改变导入、同步和后端数据库协议。
- 删除供应商后允许旧 ID 暂留本地存储；过滤只匹配当前实际存在的供应商，不影响显示。

## 非目标

- 不同步收藏到 WebDAV/S3。
- 不改变手动排序、置顶或故障转移顺序。
- 不自动把收藏项排到列表顶部。

## 兼容与迁移

- 无需数据库迁移。
- 新版本首次使用时收藏集合为空。
- 非数组或无效 JSON 会被忽略。

## 验收标准与回归清单

- [ ] 卡片左上角星标可收藏和取消收藏，且不触发拖拽或卡片其他操作。
- [ ] 「仅收藏」位于「仅显示可用」旁边，启用后只显示当前应用的收藏项。
- [ ] 收藏可跨重启保留，不同应用互不串联。
- [ ] 「清除筛选」会关闭「仅收藏」。
- [ ] 收藏筛选可与搜索、模型及可用状态组合。
- [ ] `pnpm exec vitest run src/utils/providerFavorites.test.ts` 通过。
- [ ] `pnpm exec tsc --noEmit` 通过。
