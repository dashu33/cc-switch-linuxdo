# 供应商排序、子菜单与序号

> 文档状态：最后同步 2026-07-16

## 目标

让 Provider 列表的顺序可理解、可切换并且稳定：普通新增默认追加到最下面；主菜单下方增加列表子菜单；每张 Provider 卡片显示当前列表序号。Provider 上下文快捷操作也集中到该子菜单，避免挤占应用级主工具栏。

## 用户行为

子菜单提供四种排序方式，并按应用分别记忆上次选择：

| 模式   | 行为                                     | 可拖拽 |
| ------ | ---------------------------------------- | ------ |
| 自定义 | 使用持久化 `sortIndex`，新增项追加到末尾 | 是     |
| 最新   | 按 `createdAt` 从新到旧                  | 否     |
| 最早   | 按 `createdAt` 从旧到新                  | 否     |
| 名称   | 按当前语言环境的名称升序                 | 否     |

非自定义模式只改变视图，不写回手动顺序。这样不会出现按名称显示时拖拽，却保存成另一套隐藏顺序的问题。

## 子菜单快捷操作

```text
[排序：自定义 最新 最早 名称]  |  [S3上传][S3下载] | [批量探测][搜索][定位当前]  [Provider 数量]
```

- AppSwitcher、设置、添加等应用级入口继续留在主工具栏。
- S3 上传/下载、批量探测、搜索和定位属于当前 Provider 列表，显示在 sticky 子菜单右侧。
- 操作区通过 `toolbarActions` 插槽接入，保留原有事件、loading、状态色、确认框与 tooltip。
- 子菜单保持单行且允许横向滚动；窄窗口不会换行或挤压卡片内容。
- 加载中或列表为空时仍显示子菜单和快捷操作，保留 S3 下载等入口。

## 序号

- 序号从 1 开始，跟随当前排序模式变化。
- 序号按完整列表计算；搜索过滤后保留原序号，不对搜索结果重新编号。
- 序号是展示值，不直接显示可能为空或不连续的 `sortIndex + 1`。

## 新增落底与旧数据兼容

数据库插入缺少 `sortIndex` 的 Provider 时，在同一事务内执行：

1. 按旧版可见顺序为同应用内 `sort_index IS NULL` 的历史记录补齐序号。
2. 计算当前应用的最大 `sort_index`。
3. 将新 Provider 写为下一个序号，并在缺少 `createdAt` 时补当前时间。

更新已有 Provider 时，输入缺少 `createdAt`/`sortIndex` 不会清空原值。Deep Link、Live 导入、普通表单等统一经过数据库兜底。

## 边界

- “新增默认落底”指默认的“自定义”顺序；选择“最新”后，新项按该模式自然显示在顶部。
- 显式指定插入位置的既有操作仍可保留其位置语义，例如当前复制功能可在调用侧传入明确的 `sortIndex`。
- 本特性不新增数据库列，不改变 Provider JSON 字段名。

- [ ] 自定义模式下连续新增 Provider 均出现在末尾。
- [ ] 旧库存在空 `sortIndex` 时，新 Provider 仍在所有旧项之后。
- [ ] 四种排序按钮切换结果正确，并按应用记忆。
- [ ] 非自定义模式无法拖拽，切回自定义后恢复拖拽。
- [ ] 每张卡显示从 1 开始的序号。
- [ ] 搜索过滤不会重排或重编号剩余结果。
- [ ] 五个 Provider 快捷按钮显示在子菜单，主工具栏从 AppSwitcher 开始。
- [ ] 窄窗口时子菜单横向滚动且不换行。
- [ ] 加载中和零 Provider 时仍可看到快捷操作。
- [ ] 更新 Provider 不会丢失原创建时间和手动顺序。
- [ ] 中文、繁体中文、英文、日文文案完整。
