# 自用特性索引

本目录是本分支相对原作者代码所做自用特性的**行为与定位**单一事实来源，用于升级主程序时逐项核对、迁移和回归。

> 完整方案见 [自用特性保存方案](./自用特性保存方案/) 与根目录 `AGENTS.md`。  
> 四层真相源：行为（本目录 README） / 定位（CODEMAP） / 代码（生产路径唯一实现） / 变更（按特性拆分的 Git 提交，可选 `patches/` 灾备）。  
> **禁止**把生产源码复制进 `自用特性/`。  
> **发布** = 推 GitHub 分支 + personal 远端包（Windows + mac 无签名）+ 本机编译 exe 并替换安装版；详见 `AGENTS.md` 与 [自用特性保存方案](./自用特性保存方案/)。

> **主线约定**：`main` = 自用产品线；`upstream-sync` = 作者镜像；检查更新只看 `dashu33/cc-switch-linuxdo` Releases。

## 主程序更新对齐流程

1. 保存当前工作树，确认无未提交关键改动。
2. 按上表逐个阅读特性 `README.md`，确认产品契约是否仍需要。
3. 按对应 `CODEMAP.md` 检查真实文件、关键符号和接口是否被上游移动或改名。
4. 将自用提交 `rebase`（或合并）到新上游；冲突按特性逐个解决。
5. 运行各特性的验证命令与回归清单。
6. 若实现、路径或行为变化，同一任务内同步更新特性 README、CODEMAP 和本索引。
7. 如使用 `patches/` 灾备层，升级完成后按需重导并淘汰过期 patch。

## 回退某个特性

1. 用本表与提交前缀 `feat|fix|docs(自用/<中文特性名>)` 定位相关提交。
2. 评估是否可独立 `revert`；若提交已与其它改动缠绕，按 CODEMAP 手工撤出。
3. 回退后删除或标注对应特性目录，并更新本索引。

## 目录约定

```text
自用特性/
  README.md
  自用特性保存方案/
    README.md
    CODEMAP.md
  其他中文特性名/
    README.md
    CODEMAP.md
```

- 路径一律使用仓库相对路径。
- 同一特性持续复用原中文目录，不按版本重复建目录。
- 生产源码只在项目原路径；本目录只保存契约、地图与说明材料。
- i18n 特性默认覆盖 `zh`、`zh-TW`、`en`、`ja`。
- 行为不变的格式化、注释、依赖维护可以不新建特性目录。
- 推荐提交前缀：`feat(自用/<特性名>)` / `fix(自用/<特性名>)` / `docs(自用/<特性名>)`。
