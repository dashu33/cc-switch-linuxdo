# 自用特性索引

本目录是本分支相对原作者代码所做自用特性的单一事实来源，用于升级主程序时逐项核对、迁移和回归。

> 强制约束见仓库根目录 `AGENTS.md`。每个特性使用中文目录名，并至少包含 `README.md` 与 `CODEMAP.md`；生产源码保留在项目原有位置，由 CODEMAP 记录真实落点，禁止复制源码形成双份实现。

## 主程序更新对齐流程

1. 拉取或合并原作者更新前，先保存当前工作树状态。
2. 按上表逐个阅读特性 `README.md`，确认产品契约是否仍需要。
3. 按对应 `CODEMAP.md` 检查真实文件、关键符号和接口是否被上游移动或改名。
4. 合并代码后运行各特性的验证命令与回归清单。
5. 若实现、路径或行为变化，同一任务内同步更新特性 README、CODEMAP 和本索引。

## 目录约定

```text
自用特性/
  README.md
  供应商排序与子菜单/
    README.md
    CODEMAP.md
  其他中文特性名/
    README.md
    CODEMAP.md
```

- 路径一律使用仓库相对路径。
- 同一特性持续复用原中文目录，不按版本重复建目录。
- i18n 特性默认覆盖 `zh`、`zh-TW`、`en`、`ja`。
- 行为不变的格式化、注释、依赖维护可以不新建特性目录。
