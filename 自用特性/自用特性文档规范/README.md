# 自用特性文档规范

> 文档状态：最后同步 2026-07-16

## 目标

把自用分支相对原作者代码的差异按特性独立记录，确保未来拉取上游更新时能快速判断哪些行为需要保留、代码落点是否变化、应运行哪些回归测试。

## 强制规则

- 根目录 `AGENTS.md` 是执行约束。
- 所有新增或改变行为的自用特性必须在 `自用特性/<中文特性名>/` 下维护。
- 每个目录至少包含产品契约 `README.md` 和代码地图 `CODEMAP.md`。
- 新建或重命名特性时同步更新 `自用特性/README.md`。
- 生产源码留在项目原目录，CODEMAP 只记录真实路径，不复制源码。
- 旧 `FEATURE/` 不再使用，避免中英文两套目录并行漂移。

## 本次迁移

- 将原 `FEATURE/` 的 11 个特性目录迁移为中文目录名。
- 修正特性之间的相对链接与 CODEMAP 内路径。
- 更新 Windows/macOS 自用发布工作流中的文档路径。
- 将旧个人版发布说明迁移为 `自用特性/发布说明-v3.17.1-自用版.md`。
- 从 `.gitignore` 移除 `AGENTS.md`，使约束文件能随仓库提交。

## 非目标

- 不把生产源码复制进特性目录。
- 不修改原作者的通用用户手册或正式 CHANGELOG。
- 不要求纯格式化、注释或依赖维护为每次变更新建特性目录。

## 验收与回归

- [ ] 根目录存在可跟踪的 `AGENTS.md`。
- [ ] 不再存在旧 `FEATURE/` 目录。
- [ ] 所有特性子目录都以中文命名并包含 README/CODEMAP。
- [ ] `自用特性/README.md` 能链接到全部特性目录。
- [ ] 特性文档内部相对链接全部有效。
- [ ] 发布工作流不再引用旧 `FEATURE` 路径。
