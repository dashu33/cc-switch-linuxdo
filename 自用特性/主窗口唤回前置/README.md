# 主窗口唤回前置

## 目标与范围

修复 Windows 上主窗口已显示但被其它窗口盖住时，托盘「打开主界面」/二次启动/深链接唤起后**无法切回最前**、只能从任务栏点开的问题。

## 用户行为

- 主窗口在后台（未最小化、可见但被其它窗口遮挡）时：
  - 托盘菜单「打开主界面」应把主窗口提到前台并获得焦点。
  - 再次启动应用（single-instance）应同样前置主窗口。
  - 深链接唤起（需 focus 主窗口时）应同样前置。
- 主窗口隐藏到托盘后，以上路径仍应先恢复任务栏图标、再显示并前置。
- **Windows**：托盘图标**左键双击**直接打开并前置主窗口（无需再点菜单「打开主界面」）。
- **macOS/Linux**：无托盘 DoubleClick 事件；仍用菜单「打开主界面」或系统既有唤回路径，行为不受 Windows 双击改动影响。
- 不永久置顶主窗口；仅在前置失败时做短暂 TOPMOST 闪动再取消。

## 关键决策

1. 统一入口：`window_focus::show_main_window` / `restore_main_window`，避免托盘、deeplink、single-instance、轻量模式退出各自复制一套 `show+focus`。
2. Windows 强制前置：在 `set_focus()` 之后用 `SetForegroundWindow` + 必要时 `AttachThreadInput`，仍失败则短暂 `HWND_TOPMOST` 再 `HWND_NOTOPMOST`。
3. Linux 继续复用既有 `linux_fix::nudge_main_window`；macOS 继续 `apply_tray_policy`。

## 非目标

- 不做用户可配置的「始终置顶」开关。
- 不改变关闭到托盘、静默启动、轻量模式的产品语义。
- 不改任务栏闪烁策略以外的 UI。

## 兼容 / 迁移

- 无配置迁移。
- 需重新编译后端（Rust）。

## 验收标准

- [ ] 打开主窗口后切到其它全屏/置顶应用，托盘「打开主界面」能把 CC Switch 拉到最前。
- [ ] 主窗口被普通窗口盖住时，托盘唤起无需再点任务栏。
- [ ] 最小化后托盘唤起可还原并聚焦。
- [ ] 关闭到托盘后托盘唤起可显示、出现任务栏按钮并聚焦。
- [ ] 二次启动（已运行时再开一次）能前置主窗口。
- [ ] Windows 托盘左键双击可打开并前置主窗口。
- [ ] macOS 菜单「打开主界面」与 Dock 唤回仍正常（不依赖 DoubleClick）。
- [ ] 唤起后窗口**不是**永久 always-on-top。
- [ ] 轻量模式退出后主窗口可正常显示聚焦。

## 回归清单

- 托盘切换供应商、轻量模式进出。
- 关闭按钮最小化到托盘（`minimizeToTrayOnClose`）。
- 静默启动后托盘打开主界面。
