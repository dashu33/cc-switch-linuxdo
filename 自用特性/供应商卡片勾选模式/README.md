# 供应商卡片勾选模式

> 维护目录：`自用特性/供应商卡片勾选模式/`

## 目标与范围

在 Provider 列表中，**按住卡片空白区域约 0.5 秒**进入勾选模式，可多选供应商并批量删除；避免误触按钮/拖拽手柄。

## 用户行为

1. 在供应商卡片的**空白区域**按住约 480ms → 进入勾选模式，并默认勾选该卡。
2. 勾选模式下：
   - 左上角显示 Checkbox；点空白或勾选框可切换选中。
   - 顶部出现操作栏：全选当前列表 / 删除所选 / 退出。
   - 删除需二次确认（勾选「无法撤销」后才能确认）。
   - 拖拽排序、置顶、收藏入口隐藏；右键跨应用复制菜单关闭。
3. 退出方式：点「退出」、按 `Esc`、切换应用（`appId` 变化）。
4. 按住按钮、链接、输入框、拖拽手柄、勾选框等交互控件**不会**进入勾选模式。

## 关键决策

1. **长按空白**：与会话页「批量管理」按钮不同，入口是长按卡片空白，更贴近触控/密集列表。
2. **复用 `onBulkDelete`**：删除链路与「按状态清理」相同（`deleteProviders`），不新增后端命令。
3. **仅当前可见列表可勾选**：筛选/搜索变化后，不可见项自动从选中集合移除。
4. **交互命中排除**：`button/a/input/.../[data-no-long-press]/[data-provider-drag-handle]` 不触发长按。

## 非目标

- 不实现拖拽多选框框选。
- 不在勾选模式下批量切换/复制（仅删除；后续可扩展）。
- 不改后端 API。

## 验收标准

- [ ] 按住卡片空白约 0.5s 进入勾选模式，本卡已勾选
- [ ] 按住按钮/收藏/拖拽柄不进入勾选模式
- [ ] 勾选模式下可全选、多选、删除（有确认框）
- [ ] Esc / 退出按钮 / 切换应用 可离开勾选模式
- [ ] 删除后列表与 toast 与现有批量删除一致

## 回归清单

1. 普通点击卡片空白：不进入勾选（短按）
2. 长按空白 → 勾选栏出现
3. 勾选 2+ 项 → 删除 → 确认 → 消失
4. 勾选模式下拖拽排序不可用
5. 中/英/日/繁文案正常
